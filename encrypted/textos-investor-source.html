Core Architecture Decision

Push vs. Pull — The V1 Strategy

TextOS doesn't replace platforms like Audible or Spotify — it publishes TO them. This is the critical insight that makes V1 achievable immediately and removes all competitive friction.

The Principle:

"TextOS is where you create. Everywhere else is a destination." The platforms become output channels, not competitors. This means V1 delivers massive value without needing to replicate any platform's features or negotiate API partnership agreements.

📤 PUSH — User → Platform (V1, Day One)

User creates in TextOS → TextOS publishes via API to destination. Easy. No platform replacement needed.

  • Blog post → WordPress, Substack, Medium, GoDaddy
  • Social content → Instagram, Facebook, LinkedIn, X
  • Audiobook → Audible, Spotify, Apple Podcasts
  • Book → Amazon KDP, Gumroad
  • Product listing → Etsy, Amazon, eBay
  • Business info → Google Business Profile, Yelp
  • Event → Eventbrite, Facebook Events
  • Menu update → Google Maps, DoorDash, Yelp
  • Job posting → Indeed, LinkedIn Jobs
📥 PULL — Platform → User (V2+)

Platforms send data back. Requires deep API integration + platform cooperation. V2+ after model is proven.

  • Audible royalty reports → text thread
  • Social media analytics → weekly summary
  • Etsy orders + reviews → instant notifications
  • Google Business reviews → reply by text
  • Event RSVPs → live count in thread
  • Blog comments → moderate by text
  • Podcast listener stats → monthly digest
  • Amazon KDP sales → earnings summary

Decision Log

Key Product & Business Decisions

Every significant decision logged with date and rationale. Investors want founders who think clearly and track their reasoning.

DateDecisionRationale
2026-04-26Brand: TextOS over HypeKit"Text" is the mechanic. "OS" signals platform-level infrastructure. HypeKit has perception and trademark conflicts. TextOS is clean, owned, and perfectly on-brand.
2026-04-26Phone number = identity. No login system.2FA works because phone numbers are globally unique. TextOS removes the "first factor" — the text IS the authentication. Eliminates the #1 SaaS dropout moment.
2026-04-26Paywall-first development disciplineRob's pattern has been to build without monetization gates. TextOS breaks this — every feature placed in a tier before it ships. Agent standing rule: always ask tier placement before building.
2026-04-26Pricing: Free / $29 Pro / $79 BusinessSubdomain (free) → custom domain ($29) follows proven Squarespace/Carrd conversion path. Push destinations gate by tier: Free = TextOS only, Pro = 3 platforms, Business = unlimited.
2026-04-26Tech stack: Node + Twilio + Cloudflare + SupabaseCost-optimized for SaaS margins. AI writes this stack natively. Serverless/edge scales without ops overhead. No SQL Server licensing.
2026-04-26Lead vertical: local service businessesMassive underserved market. Real pain (2014 websites). Word-of-mouth converts. Clear ROI. Easy demo story. High willingness to pay $29/mo.
2026-04-26Two modes: Unidirectional V1 → Omnidirectional V2+Start solo (no network effect required). Bridge existing platforms in V2. Native social after critical mass. Camille Gaudet contributed platform fatigue framing.
2026-04-26Fake profile resistance as trust architecture"Want a fake profile? Buy another phone." Phone-number identity makes TextOS communities inherently more authentic. Camille Gaudet insight.
2026-04-26Unidirectional Push architecture — TextOS publishes TO platforms, doesn't replace themRob's insight: pushing content to Audible/Etsy/Spotify is easy via API. Pulling data back is V3. V1 delivers massive value without competing with existing platforms. TextOS = universal creator hub. Platforms = output destinations.
2026-04-26Pricing tier update: push destinations gate by tier$19.99/mo Starter → TextOS site only. Pro ($29) → 3 external platforms. Business ($79) → unlimited. Founding members: $9.99/mo forever. Creates natural upgrade pressure as users want to publish to more destinations.
2026-04-26Identity layer moved to V1a — not V3Identity is unidirectional push — same mechanic as publishing. Self-interest is the strongest motivator. Everyone has an identity; not everyone has content ready. Zero barrier entry. Sells the now (one place for your identity) AND the promise (propagates everywhere someday). Roadmap revised: V1a=Identity, V1b=Push to TextOS site, V2=Push to external platforms, V3=Pull, V4=Omnidirectional.
