Module M-01 — The Shift

How Building Software Changed

2022 — The Old Way

  • Hire a developer or learn to code yourself
  • Months building basic infrastructure
  • $50K–$200K before your first user
  • Technical debt from day one
  • Great ideas die in the vision-to-execution gap

2026 — The Vibe Coding Way

  • Describe the outcome. AI writes the code.
  • Production infrastructure in hours
  • $0–$500 before your first user
  • Architecture stays in your head and docs
  • Build speed limited by clarity, not code
⚠️

Module M-01 — Core Principles

The 3 Principles That Make It Work

01

Architectural Thinking

You must understand what you're building well enough to describe it precisely. The more clearly you think, the better the AI output. Clarity is the skill.

02

Always understand the existing system before changing it. The biggest mistakes come from skipping research and guessing at solutions.

03

AI claims it's done. Often it isn't. Verify in the browser, logs, and database — not just by reading the AI's response.

Module M-02 — The Request Flow

How a Website Actually Works

Every time you visit a website, a small miracle happens in under a second.

🧑‍💻
Step 1
You Type a URL
A human-readable address — like a street address for a computer.
→
📖
Step 2
DNS Lookup
Translates the name to an IP address. The internet's phone book.
→
🌐
Step 3
Request Travels
Hops across the internet, reaching the nearest Cloudflare server.
→
🖥️
Step 4
Server Responds
Sends back files: HTML, CSS, and JavaScript.
→
🎨
Step 5
Browser Renders
Reads the files and paints what you see. Under 200ms start to finish.
HTML — Structure

The skeleton. Headings, paragraphs, buttons, images. Like the walls and rooms of a building.

CSS — Style

The paint and furniture. Colors, fonts, spacing, layout. Makes things look the way they look.

JavaScript — Behavior

The light switches and elevators. Handles clicks, loads data, makes pages interactive.

Six distinct layers. Each does a specific job. When you write a brief, you're targeting exactly one of these.

Supabase / Postgres

Cloudflare Pages

When someone visits your app, the very first computer they reach is a Cloudflare server. That's intentional, and it's a superpower.

🌐

Global Edge Network

300+ data centers. A user in Tokyo hits a Cloudflare server in Tokyo, not Virginia. Sub-100ms load times anywhere on earth.

📄

Pages — Static Hosting

Your HTML/CSS/JS deploys to every edge location automatically. Free SSL. One command puts your app live globally in 60 seconds.

⚡

Workers — Serverless Backend

Run backend code at the edge — no server to manage, no server bill. Zero cold start time.

🛡️

Security — Free

DDoS protection, SSL/TLS, WAF, bot filtering — all automatic and free. Enterprise security that costs thousands elsewhere.

✅

Module M-05 — The Most Important Concept

The Two Roles — Architect vs. Implementer

You — via Claude.ai
The Architect
  • Write briefs specifying WHAT to build and WHY
  • Define constraints and patterns to follow
  • Reference existing working code as the pattern
  • Make all product decisions
  • Decide when something is good enough to ship
🚫 NeverWrite specific function bodies, line-by-line fixes, or paste code blocks in your briefs.
Claude Code — AI's Role
The Implementer
  • Figures out HOW to implement your brief
  • Writes and tests the actual code
  • Deploys to the test environment first
  • Reports what was done and what to verify
  • Never makes product decisions on its own
🚫 NeverImplement anything without understanding the existing architecture first.

Module M-06 — Good vs. Bad Brief

Side by Side — What Good Looks Like

❌ Bad Brief

  • "Make it look better" — no outcome, no files, no standard
  • "Fix the login page" — which problem? What does fixed mean?
  • "Add this code: [pastes 40 lines]" — you are the implementer now
  • "Make it like Stripe's dashboard" — no pattern in your codebase
  • "Do whatever you think is best" — no constraints = no control
💸

These cost you hours of back-and-forth and rework.

✅ Good Brief

  • WHAT: Add a success toast notification after form submit
  • FILES: src/pages/contact.astro only
  • PATTERN: Same toast as dashboard.astro line 234
  • DO NOT: Touch any other page or shared component
  • VERIFY: Submit form on test URL, see green toast for 3s
⚡

Done right in one pass. Every time.

Test Environment
your-app-test.pages.dev

  • No real charges, no real user data
  • Stripe in test mode (fake payment cards)

Afternoon — A-02

The Debugging Brief Formula

STEP 1 — DO NOT FIX ANYTHING YET Read the exact error message. Paste it here in full: [error text] Read the file where the error occurs. Show me lines X through Y. STEP 2 — TRACE THE CODE PATH What function called what function that caused this error? Show me the exact call chain, step by step. STEP 3 — SEARCH BEFORE FIXING Search the entire codebase for the same pattern. How many places have the same potential problem? STEP 4 — ONLY THEN APPLY A FIX Fix the root cause — not the symptom. Show me the exact diff before applying anything. Deploy to test and verify the error is completely resolved.
📖

Real story: A 7-hour session came from skipping Step 1. We deployed 15+ times guessing. The answer was one line — documented in CLAUDE.md. Research before fix. Every time.
