## Architecture Brief for AI Implementation Agent

## WHAT IS ALREADY BUILT

TextOS can currently:
- Accept a business concept + mission statement from an activator (human or AI-generated)
- Generate a business concept autonomously when no input is provided
- Operate an AI agent layer that runs 24/7

**The next phase builds everything that comes after the mission is defined.**

---

## THE FULL AUTONOMOUS BUSINESS LOOP

This is the complete pipeline TextOS must execute without human involvement after activation.

```
PHASE 1 — FOUNDATION (Days 1–3)
  Mission defined (by activator or auto-generated by TextOS)
         ↓
  Product/service built and hosted
         ↓
  Website with paywall deployed
         ↓
  Payment processing connected (Stripe)

PHASE 2 — ACQUISITION (Days 3–7)
  Ad creative generated (copy + images + video scripts)
         ↓
  Ad campaigns published via platform APIs
  [Meta · Google · TikTok · LinkedIn · Reddit · Pinterest · X]
         ↓
  Cold traffic hits the paywall
         ↓
  Users pay → revenue collected

PHASE 3 — OPTIMIZATION (Ongoing, automated)
  Performance data ingested from ad platforms
         ↓
  Winning creatives scaled, losing ones killed
         ↓
  New creative variants generated and tested
         ↓
  Product improved based on user behavior + feedback
         ↓
  Revenue reinvested into next ad cycle (configurable %)
```

### 1.2 — Product Build Pipeline

For each business type, the agent executes a defined build sequence. Below is the universal sequence with type-specific variations noted.

**Step 1: Generate Product Assets**

The agent generates all content required for the product based on the mission and type.

```
INPUT:  mission_statement, business_type, target_audience
OUTPUT: product_assets{}

product_assets = {
  "headline":         [primary value proposition, max 10 words],
  "subheadline":      [supporting statement, max 20 words],
  "product_name":     [brand name for the product/service],
  "product_description": [250-word description for sales page],
  "feature_list":     [5–7 bullet points of what they get],
  "pricing":          [recommended price points with rationale],
  "lead_magnet":      [free offer to capture email before paywall],
  "about_us":         [50-word brand voice statement],
  "faq":              [5 most likely objections + answers],
  "legal":            [privacy policy + terms of service, auto-generated]
}
```

For content-based types (NEWSLETTER, DIGITAL_PRODUCT, COURSE, DATA_PRODUCT):
- Agent also generates the first full unit of the product
- Newsletter: First 3 issues written and queued
- Digital product: Full PDF/template generated
- Course: Full curriculum outline + first 3 lesson scripts
- Data product: First data set collected and structured

---

**Step 2: Build the Website**

The agent builds and deploys a complete marketing website and product delivery system.

```
REQUIRED PAGES:
  / (landing page)         — hero, features, social proof, CTA, pricing, FAQ
  /signup                  — email capture / account creation
  /checkout                — Stripe checkout integration
  /dashboard               — logged-in product delivery area
  /confirm                 — post-purchase confirmation + onboarding
  /privacy                 — auto-generated policy
  /terms                   — auto-generated terms

TECH STACK (existing TextOS infrastructure):
  Frontend:   Astro (static pages) deployed to Cloudflare Pages
  Backend:    Cloudflare Workers (API routes, webhooks)
  Database:   Supabase (users, subscriptions, content)
  Auth:       Supabase Auth (email + social login)
  Payments:   Stripe (subscriptions, one-time, webhooks)
  Email:      SendGrid or Resend (transactional + sequences)
  Domain:     Auto-provisioned subdomain on textos.ai or custom domain
```

Design rules:
- Use a proven high-conversion landing page template
- Above-the-fold must contain: headline, subheadline, single CTA, social proof signal
- Pricing must be visible without scrolling on desktop
- Mobile-first: every page tested at 390px before launch
- No placeholder content — every field populated from product_assets before deploy

## PHASE 2 — AD CREATIVE GENERATION

### 2.1 — Creative Assets the Agent Must Generate

For each campaign, the agent generates a full creative suite before publishing.

**Text Assets:**
```
primary_headlines:      [10 variants — problem-focused, benefit-focused, curiosity]
body_copy_short:        [10 variants — 125 chars for Meta/TikTok]
body_copy_long:         [5 variants — 500 chars for Google Display/LinkedIn]
cta_variants:           [10 CTA button texts — beyond "Buy Now"]
social_proof_copy:      [5 variants using generated or seeded testimonials]
objection_busters:      [5 variants addressing the top purchase objections]
```

**Image Ad Assets:**
```
static_image_concepts:  [10 art direction briefs for image generation]
  — Each brief includes: visual concept, text overlay, color palette, mood
carousel_concepts:      [3 carousel series, 3–5 slides each]
  — Format: problem frame → solution reveal → proof → CTA
```

**Video Ad Assets:**
```
video_scripts:          [5 scripts at 15s, 30s, 60s lengths]
  — Script format:
    HOOK (0–3s):    pattern interrupt — stop the scroll
    PROBLEM (3–8s): name the pain clearly
    SOLUTION (8–20s): show/tell the product
    PROOF (20–25s): social proof signal
    CTA (25–30s):   single clear action
ugc_style_scripts:      [3 scripts written as if from a real user]
talking_head_prompts:   [5 AI avatar or stock footage briefs]
```

**Platform-Specific Variants:**
The agent generates format variants for each platform's spec requirements.

```
META:       1:1 (1080×1080), 4:5 (1080×1350), 9:16 (1080×1920), 1.91:1 (1200×628)
GOOGLE:     Responsive Display (headlines, descriptions, images to spec)
TIKTOK:     9:16 vertical only (1080×1920)
LINKEDIN:   1.91:1 (1200×627), 1:1 (1200×1200)
PINTEREST:  2:3 (1000×1500)
REDDIT:     16:9 (1200×628)
```

### 2.2 — Ad Campaign Structure

Each business launch uses this default campaign structure, auto-configured by the agent.

```
CAMPAIGN STRUCTURE:

Campaign 1: COLD ACQUISITION
  Ad Set A: Broad interest targeting (largest audience)
  Ad Set B: Lookalike audiences (seeded from email list if available)
  Ad Set C: Keyword/search intent targeting
  
  Budget split: 60% to best performing ad set after 72 hours
  Creative: 3–5 ad variants per ad set
  Objective: Conversion (paywall event) or Lead (email capture)

Campaign 2: RETARGETING
  Ad Set A: Website visitors (last 30 days) who did not purchase
  Ad Set B: Lead magnet downloaders who did not upgrade
  Ad Set C: Checkout abandoners
  
  Budget: 20% of total daily budget
  Creative: Urgency + social proof variants
  Frequency cap: 3 impressions per user per day

Campaign 3: RETENTION (post-launch)
  Ad Set A: Existing free tier users — push to paid
  Budget: 10% of total daily budget
  Delivery: Only to custom audience of known users
```

### 2.3 — Ad Platform API Integration Requirements

The agent must be able to connect to and publish campaigns via the following APIs. Each requires credentials stored in environment variables and managed securely.

```
META MARKETING API (Facebook + Instagram)
  Endpoint:     graph.facebook.com/v19.0
  Auth:         System User Access Token (long-lived)
  Capabilities: Create campaigns, ad sets, ads, upload creatives,
                read performance metrics, manage budgets
  Key scopes:   ads_management, ads_read, business_management

GOOGLE ADS API
  Endpoint:     googleads.googleapis.com
  Auth:         OAuth2 + developer token
  Capabilities: Search, Display, YouTube, Shopping campaigns
  Key objects:  Campaign, AdGroup, Ad, BiddingStrategy, Budget

TIKTOK FOR BUSINESS API
  Endpoint:     business-api.tiktok.com/open_api/v1.3
  Auth:         App access token
  Capabilities: Campaign creation, video ad upload, audience targeting

LINKEDIN MARKETING API
  Endpoint:     api.linkedin.com/rest/adAccounts
  Auth:         OAuth2 with r_ads + w_ads scopes
  Capabilities: Sponsored Content, Message Ads, Lead Gen Forms

REDDIT ADS API
  Endpoint:     ads-api.reddit.com/api/v2
  Auth:         OAuth2
  Capabilities: Campaign creation, subreddit targeting, promoted posts

PINTEREST ADS API
  Endpoint:     api.pinterest.com/v5/ad_accounts
  Auth:         OAuth2
  Capabilities: Promoted Pins, Shopping ads, video ads
```

**Credential Management Rules:**
- All API keys stored in environment variables — never in code or logs
- Credentials rotated every 90 days by the agent
- If any API call returns 401, agent pauses that platform and alerts dashboard
- Never display credential values in any output or log

## PHASE 3 — AUTONOMOUS OPTIMIZATION ENGINE

### 3.1 — Performance Data Ingestion

The agent pulls performance data from all ad platforms on a defined schedule.

```
INGESTION SCHEDULE:
  Every 6 hours:   Pull campaign-level spend, impressions, clicks, conversions
  Every 24 hours:  Pull ad-level creative performance data
  Every 7 days:    Full audit — pull audience insights, frequency data, placement breakdown

KEY METRICS TRACKED PER AD:
  CTR (click-through rate)      — benchmark: >1.5% on Meta, >3% on Google Search
  CPC (cost per click)          — alert if 2x above campaign average
  CPL (cost per lead)           — target set at launch based on LTV model
  CPA (cost per acquisition)    — primary optimization metric
  ROAS (return on ad spend)     — target: minimum 2.0x, scale above 3.0x
  Frequency                     — alert at 4+ for cold audiences (creative fatigue)
  Relevance/Quality Score       — pull from each platform's native scoring
```

---

### 3.2 — Optimization Decision Rules

The agent makes budget and creative decisions automatically based on these rules.

```
BUDGET REALLOCATION (runs every 24 hours):
  IF ad_set ROAS > 3.0 AND spend > $50:
    → Increase daily budget by 20%
  IF ad_set ROAS < 1.5 AND spend > $30:
    → Decrease daily budget by 50%
  IF ad_set ROAS < 0.8 AND spend > $50:
    → Pause ad set, generate replacement brief

CREATIVE ROTATION (runs every 72 hours):
  IF ad CTR < 0.8% AND impressions > 2000:
    → Mark ad as underperforming
  IF ad frequency > 4.0:
    → Mark ad as fatigued
  IF 3+ ads in ad set are underperforming or fatigued:
    → Generate 5 new creative variants from winning patterns
    → Launch A/B test with 20% of ad set budget

CREATIVE LEARNING LOOP:
  Agent identifies the top 3 performing ads by ROAS each week
  Extracts: hook type, visual style, copy angle, CTA used
  Generates new variants that combine elements of winners
  Tags all creatives with performance data for future pattern matching

BUDGET SCALING GATES:
  Week 1:   Run at configured daily budget — data gathering only
  Week 2:   Scale winning ad sets by up to 30% if ROAS > 2.5
  Week 3+:  Scale by up to 50% per week if ROAS remains above target
  Hard cap: Never exceed 3x the activator's configured monthly budget
            without explicit approval
```

### 3.3 — Product Improvement Loop

The agent uses user behavior and revenue data to improve the product automatically.

```
DATA SOURCES:
  - Stripe webhooks (purchases, upgrades, cancellations, failed payments)
  - Email sequence open/click data (via SendGrid/Resend events)
  - Supabase user behavior (feature usage, session depth, drop-off points)
  - Ad platform audience insights (demographics, interests of converters)

IMPROVEMENT TRIGGERS:
  IF monthly_churn_rate > 10%:
    → Analyze cancellation reasons (exit survey or behavioral inference)
    → Generate improved onboarding sequence
    → Identify most-used vs. least-used features
    → Deprioritize unused features in UI

  IF conversion_rate (free→paid) < 5%:
    → Generate 3 new paywall positioning variants
    → A/B test pricing page copy
    → Analyze which free tier actions correlate with paid conversion
    → Shorten or restructure free trial flow

  IF NPS_signal < 7 (inferred from retention data):
    → Generate new product content or features
    → Run re-engagement email to inactive free users

  Monthly (every 30 days):
    → Full product audit generated and stored in business dashboard
    → Recommendations ranked by estimated revenue impact
    → Top 3 recommendations auto-implemented if confidence > 80%
    → Remaining queued for activator review
```

## ACTIVATOR INTERFACE

The human activator interacts with this entire system through a single dashboard. Their only required actions:

```
AT ACTIVATION:
  1. Provide mission statement (or accept TextOS suggestion)
  2. Set monthly ad budget (minimum $300/month recommended)
  3. Connect payment method for ad spend passthrough
  4. Optionally: set target audience parameters

ONGOING (optional — all have intelligent defaults):
  - Review weekly performance summary
  - Approve or reject product recommendations above 80% confidence threshold
  - Adjust ad budget up or down
  - Pause or restart the business

THAT IS ALL.
```

Everything else in this document is executed autonomously.

## CONTENT GENERATION STANDARDS

All content generated by TextOS for these businesses must meet these standards before publishing.

```
QUALITY GATES (checked before any content goes live):
  ✓ No factual claims that cannot be substantiated
  ✓ No guaranteed income or results claims in ad copy
  ✓ Privacy policy and terms of service present on all landing pages
  ✓ Unsubscribe mechanism present in all email sequences
  ✓ Ad copy complies with platform policies (no superlatives like "best ever")
  ✓ No placeholder text (lorem ipsum, [INSERT NAME], etc.)
  ✓ Mobile rendering verified before deploy
  ✓ Page load speed < 3 seconds (Lighthouse score > 80)
  ✓ All external links open in new tab
  ✓ Stripe webhook verified before traffic is sent to paywall
```
