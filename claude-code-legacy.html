[Intro]Hey, it's Rob Gaudet. I'm a software architect — thirty years of building production systems — and for the last two years I've been doing something most people in my world haven't: going deep on live AI implementation. Not reading about it. Not consulting from a distance. Actually building things. Shipping them. Finding out what holds up. Today I want to walk you through something I set up this week that I think is genuinely important for anyone writing software right now — especially if you're working on a legacy codebase. I'm going to show you how I connected Claude Code to a real .NET WebForms application. Step by step. And I'm going to tell you why it matters. [The Problem With The Chat Window]Here's how most people use AI for development today. They open a chat window — Claude, ChatGPT, whatever — they paste in a block of code, they ask a question, they get an answer, and they paste it back. And it works. I built a significant commercial application this way. But there's a ceiling on it. The AI only knows what you paste in. It doesn't know what's in the file next to it. It doesn't know your data layer. It doesn't know the pattern you established six months ago that everything else depends on. You're always working with a partial picture. For small changes, fine. But for architecture work — for actually evolving how a system is structured — you need the AI to see the whole thing. [The Stack: Why WebForms]My project is called Reyla. It's a commercial real estate data platform — AI-assisted analysis, property comps, market data. Built in C# and ASP.NET WebForms. I know what you're thinking. WebForms. That's 2002 technology. Why? Because WebForms still runs a massive percentage of enterprise .NET applications. If you're an IT director at a mid-size company, odds are good you have WebForms running something important. And those applications need to evolve — they need AI added, they need new patterns, they need refactoring — but the advice for how to do that with AI tooling assumes you're on React or microservices. I'm showing you this on WebForms on purpose. Because if it works here, it works anywhere. [The Setup: Three Commands]Here's what I did. First, I opened a terminal. On Windows, I used PowerShell — but this works the same in WSL, Mac Terminal, or Linux. Command one: npm install -g @anthropic-ai/claude-code That installs the Claude Code CLI globally on your machine. Second, I navigated to my project: cd C:\Users\rober\source\repos\Reyla Third: claude First launch asks you to authenticate. I have a Claude Max subscription, so I logged in with my Anthropic account — usage is included in the plan. If you're on the API, you can use an API key instead. The browser opens, you authorize, you go back to your terminal, and you're in. That's the entire setup. [The First Conversation]The first thing I did was ask for an architectural overview. "Give me an architectural overview of this project." Claude read through my codebase — every file — and came back with a structured map of my application. It identified the page structure, the code-behind patterns, where my data access was happening, what my business logic layer looked like, and where the pain points were likely to be if I wanted to make structural changes. It was accurate. Not generic. Specific to my project, my patterns, my choices. That's the moment I understood what was different about this approach. [What This Unlocks]Once you have full-codebase context, the conversations you can have change completely. Instead of "here's a function, make it better," you can say: — "I want to move all data access into a repository pattern — plan that refactor for me." — "There's a performance problem somewhere in my property search — help me find it." — "I want to add an AI analysis layer to this application — where does it fit architecturally?" Claude can reason across your whole project. It knows which files would be affected, what the dependencies are, what risks exist. This is the difference between a code autocomplete tool and an actual architectural collaborator. [The Bigger Picture]Here's what I want you to take from this. The AI tools most people know about — the chat windows, the autocomplete — those are the beginning. They're useful. But there's a next level, and it's available right now, and most developers haven't made the jump yet. If you're building software — whether it's a brand new React app or a fifteen-year-old WebForms application — and you want to move faster without making a mess, this is worth your time.
