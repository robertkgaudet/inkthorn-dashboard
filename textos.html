🔑 Core Breakthrough

We Just Eliminated Usernames & Passwords.

Here's why two-factor authentication works: your phone number is globally unique. When a site texts you a code, it's trusting that you have that device in your hand. That trust IS authentication.

TextOS flips that insight into a product principle. You're not receiving a code to prove who you are — you're texting us, which already proves it. Your phone number is your account. Your device is your key. There is no first factor because you already are the factor.

This eliminates the #1 dropout moment in every SaaS product: the "create an account" screen. Username. Password. Confirm password. Verify email. Forgot password. Reset link. The gauntlet that exists only because the web was built without identity baked in. SMS was built with identity baked in.

"TextOS has no login screen — because it doesn't need one. Your number is your identity. Your phone is your key. We just killed the password."

Technology

The Full Stack

Built on modern, lean infrastructure. AI does the heavy lifting. OpenClaw is the brain. Every layer is chosen for speed, cost-efficiency, and scale.

Layer Technology Purpose
Inbound SMS Twilio Receives texts from any US phone number. One dedicated number per user, provisioned automatically at signup.
Agent Brain OpenClaw (open source) The AI operating layer — persistent memory, intent understanding, tool execution, scheduling. Each user gets their own session. TextOS is OpenClaw productized for civilians.
Content Generation Claude + Grok (xAI) Site copy, SEO metadata, taglines, blog posts, logo prompts, page content — generated and iterated on demand.
Media Generation fal.ai / Flux Logo generation (3 options on signup), hero images, site photography, social media assets — all via API, included in the flow.
Site Publishing Astro + Cloudflare Pages Static site generation. Fast, SEO-friendly, zero server cost per site. Every publish is a file write + deploy trigger.
DNS & Subdomains Cloudflare API Auto-provisions name.textos.ai on signup. Upgrades to custom domain on Pro — DNS, SSL, everything handled invisibly.
Multi-Tenant Data PostgreSQL via Supabase Users, sites, sessions, billing, site configs. Fully managed, free tier to start, scales without intervention.
Billing Stripe Subscription management. Starter → Pro → Business upgrade triggered by texting "upgrade my plan".
SEO + GEO Built into every publish Schema.org markup, OpenGraph tags, semantic HTML, llms.txt — every site is AI-search-ready by default. Not an add-on.
Backend API Node.js on Railway Thin webhook handler: Twilio inbound → route to OpenClaw agent → agent executes → site publishes. The heavy lifting is in the agent, not the API.
Why not C# / WebForms / SQL Server? Licensing kills margin on a $29/month SaaS. Supabase is free until serious scale. AI writes Node, Python, and Astro faster and better right now — and this stack is what it naturally produces. Serverless + edge (Cloudflare Workers, Pages) is where modern web publishing lives. No IIS. No VMs. No ops overhead.
