01

Overview

InkThorn is a single-tenant personal productivity application delivered as a static frontend (GitHub Pages) backed by a Cloudflare Worker API. There is no build step, no framework, and no bundler — both pages are self-contained HTML files with embedded CSS and vanilla JavaScript.

The two pages share a common auth layer (Google OAuth via the Worker) and a common Supabase-backed data store, but are architecturally independent in terms of rendering and state.

No framework policy: State mutations trigger explicit re-render calls. Bundle size is zero. Any engineer can open the file and read it top to bottom with no toolchain.

03

Authentication Flow

Both pages share an identical auth mechanism:

1. User lands on notes.html — the auth entry point
2. If no token → login overlay with "Continue with Google"
3. Click → redirect to /auth/google on the Worker
4. Worker runs OAuth 2.0 PKCE with Google → creates/retrieves Supabase user
5. Redirect to notes.html?token=<JWT>
6. Frontend strips token from URL, stores in localStorage['inkthorn_token']
7. All subsequent API calls use Authorization: Bearer <token>

daycycle.html reads the same key. If absent → immediate redirect to notes.html.
Token expiry: The JWT payload contains an exp field. On init, both pages base64-decode the payload and reject expired tokens, clearing storage and forcing re-auth.
04

notes.html — Note Management App

Auth Entry Point

4.1 Purpose & Visual Design

The primary note capture and task management interface. Users can create, categorize, prioritize, schedule, and delete notes. It also serves as the authentication entry point for the entire application.

  • Aesthetic: Classic dark editorial — deep warm blacks, gold (#c9a84c) accents, crimson red (#c0392b) for Priority A
  • Fonts: Playfair Display (headings / character voice), Inter (UI copy)
  • Brand character: InkThorn — a grumpy, dapper red lobster whose voice appears in quips, placeholders, and toast notifications

4.3 Application State

All state lives in module-scoped JavaScript variables — no framework, no store:

VariablePurpose
notesCacheIn-memory array of all notes from the API
currentUserJWT payload of the authenticated user
currentSectionActive tab: daily / ideas / projects / someday
currentViewWithin Daily: week or day
currentDateThe selected date in Day View
currentWeekOffsetWeek navigation offset from today (±n weeks)
currentPriorityFilterall / A / B / C / done
currentLayoutModeDay View layout: list or planner
isProBoolean — fetched from /api/me, gates free tier cap

4.4 Local Meta System

Supabase stores note text and category only. All task metadata (priority, done state, scheduled date, time slot) is stored client-side in localStorage under the key inkthorn_daytimer_meta.

{
  "<note-id>": {
    "priority": "A",          // "A" | "B" | "C" | null
    "done": false,
    "scheduled_date": "2026-06-01",
    "time_slot": 9            // hour integer for Planner view
  }
}
Important: Meta is device-local. Switching devices loses priority and done state. This is a known limitation, not a bug — see Section 10.

4.5–4.7 Section Navigation & Views

Four tabs all render into the same #mainContent div. renderCurrentView() branches on section and view state:

Daily + week → renderWeekView()
Daily + day → renderDayView()
Ideas → renderListView() (category: ideas)
Projects → renderListView() (category: tasks)
Someday → renderListView() (category: other)

All rendering is innerHTML assignment — no virtual DOM, no diffing. The entire view re-renders on every state change.

Week View: 7-column CSS Grid (Mon–Sun). Each column is clickable → enterDay(date) switches to Day View. A "Plan Week" banner appears on Sundays after 4pm.

Day View: Includes a carry-forward system — on today's view only, undone notes from past scheduled dates surface above today's list in a "Ledger of Unfinished Business" section. Supports priority filter tabs (A/B/C/all/done) and a layout toggle between flat list and time-block Planner grid.

4.8 API Calls

MethodEndpointPurpose
GET/api/meFetch user profile + pro status
GET/api/notesLoad all notes for the user
POST/api/notesCreate a new note
DELETE/api/notes/:idDelete a note
POST/api/disconnect-telegramUnlink Telegram account
POST/stripe/checkoutStart Stripe checkout session

All calls go through apiFetch(), which injects the Bearer token and handles 401 by clearing storage and showing the login overlay.

4.9 Real-time Sync

startLiveSync() polls /api/notes every 3 seconds. If the response differs from notesCache (via JSON string comparison), it updates the cache and re-renders. This provides rudimentary multi-tab sync.

4.10–4.11 Free Tier Cap & Upgrade

  • FREE_LIMIT = 30 notes. isPro is fetched from /api/me on init.
  • Hitting the cap opens the upgrade modal instead of creating a note.
  • startCheckout() calls POST /stripe/checkout and redirects to Stripe.
  • After payment, Stripe redirects to notes.html?upgraded=1, which triggers an InkThorn success toast.

daycycle.html — AI Day Cycle

AI-Powered

5.2 Visual Design

  • Aesthetic: Miami Synthwave — deep black/charcoal, amber (#c8832a) and cyan (#00d4ff) accents, scanline overlay, CSS grid pattern
  • Fonts: Bebas Neue (brand/headings), Space Grotesk (UI), DM Sans (fallback)
  • Responsive — collapses to single-column on screens ≤600px with hamburger menu replacing desktop tab row

5.3 Data Flow

1. On load → check localStorage['inkthorn_token']
└── absent → redirect to notes.html immediately
2. GET /api/profile → check onboarding_complete
└── false → redirect to onboarding.html
3. GET /api/cycle
├── Cache hit → return stored JSON from Supabase day_cycles
└── Cache miss → Worker calls Claude, fetches Google Places
+ Bandsintown, builds cycle JSON,
stores in Supabase, returns it
4. renderCycle(data, profile, places) → paints the full UI

5.4 Cycle JSON Structure

{
  "blocks": [{
    "time": "6:00 AM",
    "icon": "🌅",
    "label": "Wake Up",
    "tips": ["First tip (white)", "Second tip (cyan)"],
    "recommendation": {          // optional — local venue
      "name": "Venue Name",
      "address": "123 Magazine St",
      "note": "Why this spot is relevant today"
    }
  }],
  "concierge": {
    "title": "Monday Anchor — TextOS & Reyla",
    "notes": ["Note 1", "Note 2"]
  },
  "private": ["Private note for partner"],
  "_city": "New Orleans",         // Worker-injected metadata
  "_neighborhood": "Uptown",
  "_lat": 29.9245,
  "_lng": -90.1133,
  "_places": [ /* Google Places results */ ]
}

_* fields are metadata injected by the Worker for the frontend to consume — they are not rendered directly to the user.

5.5 Timeline Rendering

Each block is rendered by renderBlock(b). Block status is computed by getBlockStatus(b.time):

StatusConditionVisual Treatment
currentWithin ±35 min of nowAmber glow, lifted, pulsing dot, NOW badge
pastMore than 35 min ago35% opacity
(future)DefaultNormal card style

On load, the timeline auto-scrolls to the current (or next upcoming) block with three deferred scroll attempts at 50ms / 300ms / 800ms to handle layout paint timing. A 60-second interval re-renders the track to update current/past state as time progresses.

5.6 View Tabs

On mobile, the tab row is hidden and replaced by a hamburger (☰) that opens a bottom sheet picker (#view-overlay).

5.7 Week Strip

A 7-day strip (Mon–Sun) renders immediately on page load before API calls resolve. It is navigable via ← / → week buttons. Selecting a day:

  • Today → calls showConcierge() to load the AI view
  • Past or future day → calls showDayType('weekday' | 'weekend') with static data (AI cycle is only generated for today)
Timezone note: Date arithmetic uses local date parts (year/month/day extraction) directly to avoid UTC timezone shift bugs that arise from new Date(dateString).

5.8 Neighborhood Bar

Below the timeline, a neighborhood map + venue strip renders using a Google Maps iframe embed and Google Places results from data._places[]. Spots are deduplicated by name and rendered as tappable cards opening Google Maps search. Falls back to a single "Explore Nearby" placeholder if no places were returned.

5.9 Private Panel

The private[] array renders as a blur-redacted card intended for partner/relationship nudges generated by Claude:

  • Default state: filter: blur(8px)
  • Tap header → togglePrivate() → filter: none
  • Auto-reblurs after 15 seconds via setTimeout

Shared Patterns

6.1 Error Handling

  • apiFetch() handles 401 → clear token → show login overlay
  • API errors surface as console logs; UI degrades gracefully to empty state
  • Cycle fetch failure shows a retry link inside #cycle-container

6.2 Animations

All animations are CSS keyframes only: slideInTop, slideInDown, slideUp, fadeIn, spin, pulse-dot, blink-badge. No animation library.

6.3 Mobile Responsiveness

  • notes.html: breaks at max-width: 700px — week grid collapses to 2 columns
  • daycycle.html: breaks at max-width: 600px — hides desktop nav links, shows hamburger menu
